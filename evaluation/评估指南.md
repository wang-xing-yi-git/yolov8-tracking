## 📊 评估指标说明

| 指标 | 公式 | 含义 | 范围 |
|------|------|------|------|
| **准确率** | (TP+TN) / (TP+FP+FN+TN) | 所有判断中正确的比例 | [0, 1] |
| **精确度** (Precision) | TP / (TP+FP) | 被判为异常的对象中，实际异常的比例 | [0, 1] |
| **召回率** (Recall) | TP / (TP+FN) | 所有异常对象中，被正确检测的比例 | [0, 1] |
| **F1分数** | 2×(精确度×召回率)/(精确度+召回率) | 精确度和召回率的调和平均 | [0, 1] |

### 混淆矩阵四元素

```
                实际正常    实际异常
程序判为正常      TN         FN (漏检)
程序判为异常      FP (误报)   TP (正检)
```

**示例解读：**
```
TP (真正例) = 50      ← 正确检测为异常
FP (假正例) = 10      ← 错误检测为异常 (误报)
FN (假负例) = 5       ← 异常没检测到 (漏检)
TN (真负例) = 200     ← 正确检测为正常

准确率 = (50+200)/(50+10+5+200) = 250/265 = 0.9434
精确度 = 50/(50+10) = 50/60 = 0.8333  ← 被标为异常的，有83.33%真的异常
召回率 = 50/(50+5) = 50/55 = 0.9091   ← 所有异常的，检测出了90.91%
```

```python
    ann_file = f"{video}.annotations.json"

    results = evaluator.evaluate()

    evaluator.print_results()

# 汇总统计
total_accuracy = sum(
    float(r['评估指标']['准确率 (Accuracy)']) 
    for r in all_results.values()
) / len(all_results)
print(f"\n平均准确率: {total_accuracy:.4f}")
```

## 📝 标注技巧

不需要标注每一帧，只标注**关键帧**：

```json
{
  "video": "test.mp4",
  "sampling_rate": 5,  // 每5帧标注一次
  "frames": [
    {
      "frame_id": 1,
      "annotated": true,
      "anomalies": [...]
    },
    {
      "frame_id": 6,
      "annotated": true,
      "anomalies": [...]
    }
  ]
}
```

### 使用标注工具

推荐使用**LabelImg**或**CVAT**来标注视频：

1. 导出视频的关键帧作为图像序列
2. 使用标注工具标注bounding box
3. 记录track_id和异常标记

## 🔧 常见问题

**Q1：如何处理ID变换/追踪丢失？**
```
如果某个对象在一些帧中没有被追踪，可以：
- 跳过这些帧的该对象
- 或在标注时标记为 "lost_frame": true
```

**Q2：多个异常类别怎样评估？**
```
可以扩展评估器支持多类分类：
anomalies: [
    {"class": "speeding", "is_anomalous": true},
    {"class": "parking_violation", "is_anomalous": false},
    {"class": "wrong_lane", "is_anomalous": false}
]
```

**Q3：如何只评估特定类别（如只评估车辆）？**
```python
# 在评估器中添加过滤
def filter_class(predictions, target_class):
    return [p for p in predictions if p['class'] == target_class]
```

## 📈 结果解释

**高精确度，低召回率** → 检测器很挑剔，误报少但漏检多
- 解决：降低异常检测阈值

**低精确度，高召回率** → 检测器太灵敏，漏检少但误报多
- 解决：提高异常检测阈值

**F1分数低** → 需要重新考虑异常定义或特征提取
