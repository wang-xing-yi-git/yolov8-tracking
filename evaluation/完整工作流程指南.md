# 异常检测精度评估 - 完整工作流程

## 📋 概述

这个工作流程指导您如何系统地评估YOLOv8追踪系统中的异常检测精度。核心思路是：

```
手动标注 → 运行程序生成预测 → 用评估器计算指标 → 分析结果优化阈值
```

---

## 🔄 完整工作流程

### 步骤1: 准备测试视频和手动标注

#### 1.1 选择测试视频
- 选择1-2个**典型的测试视频**（最好包含异常情况）
- 建议长度：30秒-2分钟（约900-3600帧，易于手动标注）
- 建议包含：
  - ✅ 正常情况（车辆正常移动、行人正常行走）
  - ✅ 异常情况（车辆突跳、行人躺下）

#### 1.2 手动标注视频
使用 `evaluation/评估指南.md` 中的JSON格式标注。

**工具选择：**
- **CVAT** (最推荐): 支持视频逐帧标注，可导出格式
- **LabelImg**: 支持XML/JSON导出
- **手工JSON**: 如果只有少数几帧，直接编写JSON文件

**标注格式：**
```json
{
  "video_name": "test_video.mp4",
  "fps": 30,
  "annotations": [
    {
      "frame_id": 1,
      "objects": [
        {
          "track_id": 1,
          "is_anomalous": false,
          "reason": ""
        },
        {
          "track_id": 2,
          "is_anomalous": false,
          "reason": ""
        }
      ]
    },
    {
      "frame_id": 2,
      "objects": [
        {
          "track_id": 1,
          "is_anomalous": true,
          "reason": "车辆突跳（传送带故障）"
        }
      ]
    }
  ]
}
```

#### 1.3 保存标注文件
```bash
# 保存格式：
```

### 步骤3: 运行评估器计算精度指标

#### 3.1 使用评估器
```python
from evaluation.anomaly_evaluator import AnomalyEvaluator

# 创建评估器
evaluator = AnomalyEvaluator(
    annotations_file='evaluation/test_annotations.json',
    predictions_file='evaluation/predictions.json'
)

# 执行评估
results = evaluator.evaluate()

# 显示结果
evaluator.print_results()
evaluator.save_results('evaluation/evaluation_results.json')
evaluator.export_csv('evaluation/evaluation_results.csv')
```

#### 3.2 理解评估指标

| 指标 | 计算公式 | 含义 |
|------|--------|------|
| **TP** | - | 正确检测到的异常 |
| **FP** | - | 错误检测为异常的正常物体 |
| **FN** | - | 遗漏的异常物体 |
| **TN** | - | 正确识别的正常物体 |
| **准确率** | (TP+TN)/(总数) | 整体正确率 |
| **精确率** | TP/(TP+FP) | 检测为异常的有多少真的异常 |
| **召回率** | TP/(TP+FN) | 实际异常被检测出多少 |
| **F1分数** | 2×精确率×召回率/(精确率+召回率) | 精确率和召回率的调和平均 |

**解释例子：**
```
假设：
- TP = 8  (正确检测到的异常)
- FP = 2  (误报)
- FN = 1  (遗漏)
- TN = 189 (正确识别的正常情况)
- 总数 = 200

则：
- 准确率 = (8+189)/200 = 98.5%
- 精确率 = 8/(8+2) = 80%     (检测为异常的中有80%真的异常)
- 召回率 = 8/(8+1) = 88.9%   (88.9%的异常被检测出来)
- F1 = 2×0.8×0.889/(0.8+0.889) = 84.2%
```

### 步骤4: 分析结果并优化参数

#### 4.1 评估结果示例

```
=== 异常检测精度报告 ===

总预测数: 200
总标注数: 200
正确预测: 197

混淆矩阵:
         预测正常  预测异常
实际正常    189      2      (FP=2)
实际异常     1       8      (FN=1, TP=8)

精度指标:
  准确率 (Accuracy):  98.50%
  精确率 (Precision): 80.00%  ← 误报率偏高
  召回率 (Recall):    88.89%
  F1分数:             84.21%

按异常原因分类:
  速度过快:    TP=6, FP=1, FN=0
  躺下姿态:    TP=2, FP=1, FN=1
```

#### 4.2 问题诊断与优化

**问题1: 精确率低（FP多）- 误报太多**
```
症状: 很多正常物体被错误检测为异常
原因: 阈值太低，容易触发异常条件
解决:
  - 提高 CAR_SPEED_THRESHOLD (例如 100 → 150)
  - 提高 PERSON_ASPECT_RATIO_THRESHOLD (例如 0.5 → 0.4)
```

**问题2: 召回率低（FN多）- 漏检**
```
症状: 很多真实异常没被检测出来
原因: 阈值太高，异常条件过严格
解决:
  - 降低 CAR_SPEED_THRESHOLD (例如 100 → 80)
  - 降低 PERSON_ASPECT_RATIO_THRESHOLD (例如 0.5 → 0.6)
```

**问题3: F1分数低 - 整体性能差**
```
症状: 精确率和召回率都不理想
原因: 
  1. 检测模型质量差（检测框不准）
  2. 追踪算法问题（ID跳跃、断裂）
  3. 特征选择不当（选错速度/姿态作为异常指标）
解决:
  - 检查detect_and_trk.py中的特征计算
  - 用离线数据验证特征计算正确性
  - 考虑添加新的异常特征
```

#### 4.3 参数调整流程

```
初始值
   ↓
运行评估 → 查看精确率/召回率
   ↓
精确率>85% && 召回率>85% ?
   ├─ YES → ✅ 满意，停止
   └─ NO  → 调整参数，重复评估
            (通常需要3-5次迭代)
```

### 技巧1: 快速标注少量数据
```bash
# 如果只想评估某些关键帧（不是全部）
# 可以只标注关键帧，导入以下文件：

# evaluation/partial_annotations.json

  "video_name": "test.mp4",
  "annotations": [

    {"frame_id": 150, "objects": [{"track_id": 2, "is_anomalous": false, "reason": ""}]},
    # ... 只标注关键帧
  ]

```

### 技巧2: 多模型对比
```python
# 使用相同的测试集对比两个模型
for model_version in ['v1', 'v2']:
    # 运行model_v1和model_v2分别生成预测
    predictions_v1 = run_model('v1', test_video)
    predictions_v2 = run_model('v2', test_video)
    
    # 对比评估结果
    eval_v1 = AnomalyEvaluator('annotations.json', 'predictions_v1.json')
    eval_v2 = AnomalyEvaluator('annotations.json', 'predictions_v2.json')
    
    print(f"模型{model_version}: F1={eval_v1.results['f1']:.2%}")
```

### 技巧3: 按类别分别评估
```python
# 分别评估"车辆异常"和"行人异常"
```

## ❓ 常见问题

**Q: 需要标注所有帧吗？**
A: 不需要。标注代表性帧即可。如果有10个异常事件，标注每个事件前后的5帧（50帧总共）足够评估模型。

**Q: 一个视频足够吗？**
A: 单个视频可用于快速原型，但建议用3-5个代表不同场景的视频来评估鲁棒性。

**Q: 精确率和召回率如何平衡？**
A: 根据实际需求：
- 安全关键应用（如避免碰撞）：优先高召回率（不要漏检异常）
- 用户体验应用（如提醒）：优先高精确率（减少误报烦恼）
- 一般应用：使用F1分数平衡

**Q: 参数调整要调多少次？**
A: 通常3-5次迭代就能达到满意结果。每次调整0.5-1倍速度阈值即可。
